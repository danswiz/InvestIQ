Recursive Language Models in ADK

Google Developer Forums - Community Article Summary

Overview

Recursive Language Models (RLM) is a breakthrough agent architecture from MIT (published Dec 2025) that enables AI agents to scale to 10M+ token contexts and solve complex information-dense tasks that defeat traditional LLM architectures.

Core Innovation

Unlike standard LLMs that face context length limitations, RLMs give agents complete control over their own context through an interactive coding environment (REPL). The agent writes and executes Python code to manipulate context and delegate tasks recursively.

Key Components

  • context variable: Contains the entire working context (up to 10M+ tokens)
  • rlm_agent(query, context) function: Spawns child agents with identical architecture — the "recursive" element
  • Pre-loaded libraries: json, re, numpy, etc. for context manipulation

How It Works

  1. Agent receives a query and full context
  2. Writes Python code using context variable and available functions
  3. Executes code, analyzes output
  4. For complex tasks: delegates sub-tasks to child agents via rlm_agent()
  5. Aggregates child agent results
  6. Repeats until task complete

Two Key Capabilities

  1. Context Manipulation: Process massive contexts via code (e.g., regex searches across millions of tokens)
  2. Recursive Task Delegation: Break problems into sub-tasks handled by child agents without burning parent agent's reasoning tokens

OOLONG-Pairs Benchmark

A novel test requiring quadratic reasoning across statement pairs in massive documents (like finding contradictions across thousands of interview transcripts).

  • RLM Score: 23/100
  • Other Agents: 0/100

Implementation in Google's ADK

The author re-implemented RLM using Google's Agent Development Kit (ADK) — an open-source framework for production-ready multi-agent systems. Key benefits:

  • Event orchestration & session management
  • Built-in evaluation framework
  • Developer UI & API
  • Enterprise deployment features

Note: Required extending ADK's BaseAgent (LLMAgent was insufficient for recursive patterns).

Novel Extensions

ExtensionDescription
Lazy File LoadingInstead of loading entire context into memory, agents reference file collections (local filesystem, GCS buckets) and load contents on-demand
ParallelismConcurrent child agent execution with configurable limits (vs sequential in original paper)
Real-Time UILive event streaming for monitoring long-running tasks and validating agent approach

Bottom Line

RLM represents a paradigm shift in agent architecture: agents that code their way through problems and recursively delegate to scale indefinitely. The ADK implementation brings this research concept into production-ready enterprise tooling.
